 # 조인
    - 관계형 DB에서 가장 기본이자 중요한 기능
    - 두개 이상의 테이블을 서로 연결하여 데이터를 추출
    - 관계형 DB에서는 조인을 통해 서로 다른 테이블간의 정보를 원하는 대로 가져올 수 있음
    - 서로 다른 테이블을 연결

## 조인 시 데이터 집합 레벨의 변화
    - 조인 컬럼 deptno 기준 M 집합 ( 중복 )
    - 조인 컬럼 deptno 기준 1 집합 ( Unique, PK )
    - 조인 결과 M 집합
    - 1 : M 조인 시 결과 집합은 "M 집합의 레벨"을 그대로 유지


### SQL 조인 시 데이터 집합 레벨의 변화 - 1
    - 상품 주문 테이블 M ( 중복 )
    - 상품 테이블 1 ( Unique )
    - "M 집합인" 상품 주문 테이블의 레벨 유지

### SQL 조인 시 데이터 집합 레벨의 변화 - 2
    - 고객 테이블 1 ( Unique )
    - 고객 연락처 테이블 M ( 중복 )


### 데이터 연결 관계 이해
    - 1:1, 1:M, M:N 관계의 이해


## 조인 유형
    - Inner Join
    - Outer Join,
    - Full Outer Join

### Inner Join
    - A -> B 의 경우
    - A(Outer), B(Inner)

### Outer Join
    - Outer가 되는 테이블의 값을 살려 나간다
    - Inner에 부합되는 값이 없는 경우 NULL 값이 들어감
